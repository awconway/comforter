REMAIN External-Validation Report (In-Hospital Mortality)

Outcome: In-hospital mortality (DeathHospDisch)
Model: REMAIN logistic (uncalibrated)
Validation design: External validation by site with one temporal-validate site (Site 2) and three external-validate sites (Site 1, Site 3, Site 4).
Site 2 is treated as the development-site temporal validation (same institution, later time period), while Sites 1, 3 and 4 are external hospital hold-outs.

Analysis Summary

This report summarizes site-level and pooled performance for the REMAIN logistic model using the current external-validation predictions.

  • External validation sites: Site 1, Site 3, Site 4
  • Temporal validation site: Site 2
  • Primary outcomes shown: ROC-AUC, PR-AUC (average precision), Brier score, calibration slope/intercept.
  • Source artifacts: all files are located in publication_ready_remain_logistic.

Key Interpretation from Notes

  • Discrimination was preserved across cohorts, with ROC-AUC roughly in the 0.77 to 0.84 range, indicating that the predictor set retains useful ranking ability (risk-ordering signal) outside the development setting.
  • Calibration-in-the-large was not the main issue. Brier scores were broadly similar across sites and baseline risk differences were modest, with the largest calibration-in-the-large deviation seen in Site 4.
  • Calibration slope showed transportability instability.
    • Temporal validation (Site 2): slope < 1 (0.825), indicating too-extreme predictions and potential attenuation of predictor effects over time.
    • External validation sites: one site showed similar attenuation (Site 3, 0.789), while another showed over-strong prediction scale (Site 4, 1.218).
  • Substantive implication: Brier score appears similar across sites and does not fully reflect calibration slope heterogeneity, so absolute risk estimation may vary substantially even when average error remains comparable.
  • Consequence: This pattern supports model refinement for calibration transportability, not a simple intercept-only correction. A full re-estimation or recalibration strategy using multicentre development data is justified if the model is to be used for absolute risk prediction.
  • What refinement should target: improved transportability is primarily about stabilizing slope behavior across sites and time, not a large AUC gain. A refined model would aim for stable predictor effects, reduced local updating need, and better probability estimates while preserving discrimination.
  • Rationale for a new model vs. only recalibration: heterogeneity in slope direction across cohorts (some slopes <1, one slope >1) indicates the current coefficients are not universally transportable; single-model global recalibration is unlikely to resolve this.
  • External transportability context: this report therefore reports external-validation performance for one temporal and three external site validations to support development from a multicentre perspective; a subsequent refined model should still be benchmarked with internal-external validation.

Relevance of a New Model

The current REMAIN logistic model appears transportable for ranking risk but not fully transportable for probability calibration across all sites and over time. This supports the argument for a refined model with more stable predictor effects across settings, followed by repeat facility-level internal-external validation to test whether multicentre recalibration/ redevelopment improves transportability.
