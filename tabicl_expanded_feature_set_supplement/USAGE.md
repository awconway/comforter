# How To Use The TabICL Archive

## What this archive enables

This archive provides the exact predictor list, value encodings, TabICL settings, and executable inference code needed to generate predicted probabilities for new individuals using the 31-variable feature set.

Because TabICL is a pre-trained tabular foundation model rather than a coefficient-based regression model, prediction requires both:

1. the frozen TabICL checkpoint version; and
2. a labelled support set used to fit the model before scoring the new query records.

## Environment

From the repository root used to assemble this archive, the required dependencies were managed with `uv` and the root `pyproject.toml` / `uv.lock` files copied into `provenance/`.

## Command example

```bash
uv run python zenodo_tabicl_expanded_feature_set_supplement/inference/predict_tabicl_expanded_feature_set_from_csv.py \
  --support-csv zenodo_tabicl_expanded_feature_set_supplement/inference/support_template_expanded_feature_set.csv \
  --query-csv zenodo_tabicl_expanded_feature_set_supplement/inference/query_template_expanded_feature_set.csv \
  --output-csv predictions_expanded_feature_set.csv
```

## Output

The inference script writes one row per query record with:

- `Id` when supplied in the query CSV;
- `row_index`;
- `proba_death`; and
- `proba_survival`.


## Included serialized object

A final all-data fitted object is included at `fitted_model/tabicl_expanded_feature_set_all_data.pkl`. This object was fit on all available rows of the COMFORTER dataset with non-missing `DeathHospDisch`, using the same 31 expanded predictors documented in the archive.

If you want to use the frozen fitted object directly rather than refitting from a support CSV, load it with Python `pickle` and pass a query dataframe containing the archived 31 predictor columns in the documented order.

## Synthetic smoke test

The archive includes a standalone synthetic demo under `synthetic_demo/`. From the supplement root:

```bash
uv sync
uv run python synthetic_demo/run_synthetic_demo.py
```

This will generate:

- `synthetic_demo/synthetic_predictions_from_refit.csv`
- `synthetic_demo/synthetic_predictions_from_serialized_object.csv`

These two outputs are not expected to match exactly, because the first is generated after refitting TabICL on the small synthetic support set, whereas the second is generated by the archived all-data fitted object.
